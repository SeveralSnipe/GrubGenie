API Documentation

UserStoreRoutes

User Items

GET "/userItems/:UserId"

Retrieves items associated with a particular user.

Parameters: UserId (string)

User Stores

GET "/userStore/:UserId/:StoreId"

Retrieves stores visited by a particular user.

Parameters: UserId (string), StoreId (string)

GET "/StoreItems/:ItemId"

Retrieves stores associated with a particular item.

Parameters: ItemId (string)

VendorRoutes

Register Item

POST "/registerItem"

Registers a new item in the store.

Request Body: ExpiryDate (string), ItemMRP (number), ItemSP (number), ItemName (string), StockQuantity (number, optional), StoreId (string)

PATCH "/updateItem/:itemId"

Updates information of an existing item.

Parameters: itemId (string)

Request Body: Any combination of ItemName (string), ExpiryDate (string), StockQuantity (number), ItemMRP (number)

RegisterRoutes

Register Store

POST "/registerStore"

Registers a new store.

Request Body: GST (string), Location (string), PhoneNumber (string, optional), StoreName (string), Email (string, optional)

Register User

POST "/registerUser"

Registers a new user.

Request Body: DOB (string, optional), PhoneNumber (string), UserName (string), Email (string, optional)

CommerceNearMeRoutes

CommerceNearMeRouter

GET "CommerceNearMeRouter/AllStores"

Retrieves all store details in alphabetical order.

POST "CommerceNearMeRouter/foodNearMeDist"

Request Body:

{
    location: [latitude, longitude],
    maxDist: number, // Maximum distance from user
    sort: "dist" or "name" // Sorting result based on name or distance from user
}

Response: Store details

POST "CommerceNearMeRouter/foodNearMeItem"

Request Body:

{
    location: [latitude, longitude],
    maxDist: number, // Maximum distance from user :- default dist 5kms
    sorts: "dist" or "name" or "price",  :- default dist
    Item: "Chicken"
}

Response: Retrieves stores with specified items.
